# Expressive voice agent — agent worker

The backend half of the [expressive voice agent demo](../README.md): a Python
[LiveKit Agents](https://docs.livekit.io/agents/) worker that speaks with
[Fish Audio](https://fish.audio)'s expressive TTS. The user flips its register
(casual/professional) at runtime via an on-screen toggle (a `set_mode` RPC swaps
the SDK expressive preset), while a separate cosmetic LLM drives an on-screen mood
ring. It starts in a caller-selected preset voice, and — depending on the landing
page choice — can instead **clone the caller** from a short scripted read or
**design a brand-new voice** from a text description before the conversation
begins. Sessions run in English or Japanese (`lang` in the dispatch metadata; the
prompts, voices, and STT model all localize).

**Looking for the live voice-cloning logic?** It's two files:
[`src/voice_clone.py`](./src/voice_clone.py) is the self-contained core — a
passthrough audio tee that buffers the caller's mic while they speak, silero VAD
trimming, WAV encoding, and the Fish Audio HTTP calls (`create_voice_clone` →
`POST /model` with `train_mode=fast`, plus voice design and model deletion).
`Assistant.run_clone_first` in [`src/agent.py`](./src/agent.py) orchestrates the
in-call flow: prompt the read, run the fixed 15s capture window, upload, swap the
live TTS to the new `model_id`, and delete the ephemeral model on session end.

This directory is self-contained — you can run it on its own and point any
[compatible frontend](https://docs.livekit.io/frontends/) (or the
[`web/`](../web/) app, or telephony) at the same LiveKit project.

## Stack

- **STT + turn detection** — Deepgram Flux (`livekit-plugins-deepgram`, `deepgram.STTv2` on the `/v2/listen` API): `flux-general-en` for English sessions, `flux-general-multi` with `language_hint=["ja"]` for Japanese. Flux does turn-taking itself via native EndOfTurn / EagerEndOfTurn events (`turn_detection="stt"`); the EagerEndOfTurn signal drives preemptive generation.
- **LLM** — OpenAI `gpt-5.1` (`livekit-plugins-openai`, your `OPENAI_API_KEY`); override the model with `OPENAI_MODEL`. A separate cosmetic mood classifier runs on `MOOD_MODEL` (default `gpt-4.1-mini`).
- **TTS** — Fish Audio `s2.1-pro` (`livekit-plugins-fishaudio`)
- **VAD** — Silero VAD, kept only for interruption / barge-in handling (Flux owns turn detection)

Per-session config (chosen voice / clone flag / design description / language)
arrives as agent dispatch metadata; the expressive mode/mood system, the
clone-first flow (read a script → VAD trim → Fish `/model` upload → TTS swap →
delete on session end), the design-first flow, and the `/jp` localization are
documented in [`CLAUDE.md`](./CLAUDE.md). The code is in
[`src/agent.py`](./src/agent.py) and [`src/voice_clone.py`](./src/voice_clone.py);
preset-voice preview clips are generated by [`scripts/gen_voice_samples.py`](./scripts/gen_voice_samples.py).

## Run it standalone

```bash
cp .env.example .env.local                  # then fill in your keys
uv sync
uv run python src/agent.py download-files   # Silero VAD weights

# Talk to it in your terminal (no LiveKit server needed):
uv run python src/agent.py console

# Or register as a worker against your LiveKit project (for use with a frontend):
uv run python src/agent.py dev
```

In production use `start` instead of `dev` (this is what the Dockerfile runs;
the demo deploys it to [LiveKit Cloud Agents](https://docs.livekit.io/agents/ops/deployment/)
via `lk agent deploy` — see the [root README](../README.md#deploy)).

### Docker

```bash
docker build -t voice-clone-agent .
docker run --rm --env-file .env.local voice-clone-agent
```

The worker has no inbound port — it connects out to LiveKit and waits for room
dispatch.

## Environment variables

```env
LIVEKIT_URL=wss://<your-project>.livekit.cloud
LIVEKIT_API_KEY=
LIVEKIT_API_SECRET=

DEEPGRAM_API_KEY=
OPENAI_API_KEY=
FISH_API_KEY=
```

Fish reads `FISH_API_KEY` (not `FISH_AUDIO_API_KEY`).
